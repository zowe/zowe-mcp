# Remote dev (Keycloak + HTTP MCP)

- **Docs:** [`docs/remote-dev-keycloak.md`](../../docs/remote-dev-keycloak.md) — includes **mock** defaults and **real z/OS (native SSH)** with JWT (`--http --native`, config, passwords).
- **Compose:** `docker-compose.yml` — Keycloak + realm bootstrap (**`keycloak-init`** is a small **Debian bookworm-slim** image built from **`Dockerfile.keycloak-init`** with **curl** and **jq** preinstalled so bootstrap can enable **OIDC Dynamic Client Registration** for local dev (avoids Alpine **`apk`** TLS fetches to `dl-cdn.alpinelinux.org`); see **`docs/remote-dev-keycloak.md`** — *Dynamic Client Registration*); optional **`zowe-mcp`** service with `--profile mcp-image` (requires `zowe-mcp-server-*.tgz` from `npm run pack:server`; image is **mock-only** — native z/OS needs a custom image or run MCP on the host; see doc). Optional **`KEYCLOAK_HOST_PORT`** (default **18080**) maps the host port to Keycloak’s container port **8080**. Commented samples for **`KC_LOG_LEVEL`** / **`ZOWE_MCP_LOG_LEVEL`** (debug) are in the file; see **`docs/remote-dev-keycloak.md`** — *Debug logging*.
- **One-command host MCP (native z/OS):** **`npm run start:remote-https-dev-native-zos`** (`scripts/start-remote-https-dev-native-zos.sh`) — see **`docs/remote-dev-keycloak.md`** and **`docker/remote-https-dev/README.md`**. After the realm is ready, **`patch-keycloak-mcp-dev-redirects.mjs`** merges Inspector-friendly **OAuth redirect URIs** and **web origins** on client `demo` (see **`docs/remote-dev-keycloak.md`** — *Browser OIDC and redirect URIs*).
- **Clean Keycloak DB (optional):** **`npm run keycloak:dev-fresh`** (native HTTPS stack — same compose as **`start:remote-https-dev-native-zos`**; runs **`keycloak-init`**, OIDC wait, then **`patch-keycloak-mcp-dev-redirects.mjs`**) — see **`docs/remote-dev-keycloak.md`** (*Stale OAuth clients*).
