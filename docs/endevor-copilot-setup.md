# Endevor Tools â€“ Setup Guide for VS Code Copilot

This guide explains how to make the Endevor CLI bridge MCP tools available in VS Code Copilot (or any MCP client). The Endevor tools wrap the [Broadcom Endevor Plug-in for Zowe CLI](https://www.npmjs.com/package/@broadcom/endevor-for-zowe-cli) and expose Endevor operations (list environments, list elements, print element source, query components, etc.) as MCP tools that AI agents can call.

## Prerequisites

1. **Zowe CLI** installed and on `PATH`:

   ```bash
   npm install -g @zowe/cli
   ```

2. **Endevor Plug-in** installed:

   ```bash
   zowe plugins install @broadcom/endevor-for-zowe-cli
   ```

3. Either a configured **Zowe Endevor profile** (recommended) or explicit host/port/credentials for the Endevor Web Services (EWS) REST API.

4. The Zowe MCP server tarball or `@zowe/mcp-server` package available (see [roo-or-standalone-mcp.md](roo-or-standalone-mcp.md) for install options).

## Configuration via `.vscode/mcp.json`

> **Note**: The Zowe MCP VS Code extension can also be used to activate Endevor tools via the `zoweMCP.cliPlugins` VS Code setting â€” see [VS Code Extension: cliPlugins setting](#vs-code-extension-cliplugins-setting) below. The `.vscode/mcp.json` approach gives you full control of the server arguments and works with any MCP client.

Create or edit `.vscode/mcp.json` in your workspace root:

### Option A â€” Profile-based (recommended)

Use an existing Zowe Endevor profile and optional location profile:

```json
{
  "servers": {
    "zowe-endevor": {
      "type": "stdio",
      "command": "node",
      "args": [
        "/path/to/zowe-mcp-server/dist/index.js",
        "--stdio",
        "--cli-plugin-yaml", "/path/to/zowe-mcp-server/dist/tools/cli-bridge/endevor-tools.yaml",
        "--cli-plugin-connection-file", "${workspaceFolder}/.endevor-connection.json"
      ]
    }
  }
}
```

Create `.endevor-connection.json` in your workspace (add to `.gitignore`):

```json
{
  "pluginParams": {
    "pluginProfile": "my-endevor-profile",
    "locationProfile": "my-location-profile"
  }
}
```

The `pluginProfile` corresponds to an `endevor` type Zowe profile (contains the EWS host/port/credentials). The `locationProfile` corresponds to an `endevor-location` type Zowe profile (contains the default environment, stage, system, subsystem, type, and instance/datasource).

### Option B â€” Explicit credentials (no profiles)

Pass all connection parameters directly in the connection JSON:

```json
{
  "pluginParams": {
    "instance": "ENDEVOR"
  },
  "host": "ews.example.com",
  "port": 8080,
  "user": "USERID",
  "password": "secret",
  "protocol": "http",
  "basePath": "EndevorService/api/v2"
}
```

> **Security note**: Avoid committing credentials to version control. Use OS secret injection or an `.env` file (not committed) and reference env vars from your shell profile.

### Option C â€” Profile-based, using the VS Code extension's bundled server

When the Zowe MCP VS Code extension is installed, the server is bundled inside the extension. Point to the bundled server:

```json
{
  "servers": {
    "zowe-endevor": {
      "type": "stdio",
      "command": "node",
      "args": [
        "${env:HOME}/.vscode/extensions/zowe.zowe-mcp-vscode-<version>/server/index.js",
        "--stdio",
        "--cli-plugin-yaml", "${env:HOME}/.vscode/extensions/zowe.zowe-mcp-vscode-<version>/server/tools/cli-bridge/endevor-tools.yaml",
        "--cli-plugin-connection-file", "${workspaceFolder}/.endevor-connection.json"
      ]
    }
  }
}
```

Replace `<version>` with the installed extension version.

## Description Variant

Three description styles are available for the Endevor tools:

| Variant | Description | Best for |
| --- | --- | --- |
| `intent` | Intent-centric, human-readable rewrites | AI agents (default) |
| `cli` | Verbatim Zowe CLI help text | Reference / debugging |
| `optimized` | LLM-optimized, generated by the rephrase script | A/B testing |

The default is `intent`. To switch, add `--cli-plugin-desc-variant <name>` to your MCP server args, or set the environment variable `ZOWE_MCP_CLI_DESC_VARIANT=optimized`.

Example for `optimized`:

```json
"args": [
  "...",
  "--cli-plugin-yaml", "/path/to/endevor-tools.yaml",
  "--cli-plugin-connection-file", "/path/to/.endevor-connection.json",
  "--cli-plugin-desc-variant", "optimized"
]
```

## Using Endevor Tools in Copilot

### 1. Set the Endevor context (once per session)

Before using most Endevor tools, tell the agent the default Endevor location. This avoids repeating environment/system/subsystem on every call:

> *"Set Endevor context to environment DEV, stage 1, system SYS1, subsystem SUB1, type COBPGM."*

This calls `endevorSetContext` which stores the location for all subsequent tool calls.

### 2. List elements

> *"List all COBOL elements in the current Endevor context."*
> *"List all elements in PRD environment, stage 2, system SYS1, subsystem SUB1, type COBPGM."*
> *"List elements matching PROG* in Endevor."*

### 3. Read element source

> *"Show me the source code of PROG01."*
> *"Print the COBOL source of the CUSTMAIN element."*

### 4. Query dependencies

> *"What does PROG01 depend on? Show all its components."*

### 5. List packages

> *"List all Endevor packages with status INENDEVOR."*

## Available Endevor Tools

| Tool | Description |
| --- | --- |
| `endevorSetContext` | Sets the default Endevor location context (environment, stage, system, subsystem, type) for the session |
| `endevorListEnvironments` | Lists all available Endevor environments |
| `endevorListStages` | Lists stages configured for a given environment |
| `endevorListSystems` | Lists systems within an environment and stage |
| `endevorListSubsystems` | Lists subsystems within a system |
| `endevorListTypes` | Lists element types within a location |
| `endevorListElements` | Lists elements matching a location (supports wildcards) |
| `endevorPrintElement` | Retrieves the source code of an element |
| `endevorListProcessorGroups` | Lists processor groups for a type |
| `endevorQueryComponents` | Queries the dependencies of an element |
| `endevorListPackages` | Lists Endevor packages (change bundles) |

In Copilot, tool names are prefixed: `mcp_zowe-endevor_endevorListElements` (using the server name from `.vscode/mcp.json`).

## Workflow Example

```
User: "Show me the source of CUSTMAIN in PRD"

Agent calls:
  1. endevorSetContext(environment=PRD, stageNumber=2, system=SYS1, subsystem=SUB1)
  2. endevorListElements(element=CUSTMAIN, type=COBPGM)
  3. endevorPrintElement(element=CUSTMAIN)

Agent returns: [source code of CUSTMAIN]
```

## Troubleshooting

**Tools not appearing in Copilot**

- Check that `.vscode/mcp.json` has no JSON syntax errors.
- Run **MCP: List Servers** from the Command Palette to see if the server started.
- Select the server â†’ **Show Output** to see startup errors.

**"Command syntax invalid" errors**

- Verify that the Endevor plugin is installed: `zowe plugins list`.
- Confirm the profile name exists: `zowe profiles list endevor`.
- Try `zowe endevor list elements * --endevor-profile <profile> --instance <instance> --rfj` in a terminal to verify the CLI works outside of MCP.

**Authentication errors**

- Check credentials in your profile or connection JSON.
- For password expiry, update via `zowe profiles update endevor <name> --password <new>`.

**Connection file not found**

- Use absolute paths in `.vscode/mcp.json` or `${workspaceFolder}/...` for workspace-relative paths.

## VS Code Extension: cliPlugins setting

The Zowe MCP VS Code extension supports a `zoweMCP.cliPlugins` array setting that activates CLI plugin tools directly through the extension-managed server â€” no `.vscode/mcp.json` needed.

Open VS Code **Settings** (<kbd>Cmd+,</kbd> / <kbd>Ctrl+,</kbd>), search for **Zowe MCP CLI Plugins**, and add entries to the JSON:

```json
"zoweMCP.cliPlugins": [
  {
    "yaml": "/absolute/path/to/endevor-tools.yaml",
    "connectionFile": "/absolute/path/to/.endevor-connection.json",
    "descVariant": "intent"
  }
]
```

| Field | Required | Description |
| --- | --- | --- |
| `yaml` | Yes | Absolute path to the plugin YAML file |
| `connectionFile` | No | Absolute path to a `CliConnectionConfig` JSON file |
| `descVariant` | No | Description variant: `cli`, `intent` (default), or `optimized` |

The connection JSON uses the same format as described in [Option A / B above](#option-a--profile-based-recommended).

After saving settings, run **Developer: Reload Window** to restart the Zowe MCP server with the plugin tools active.
